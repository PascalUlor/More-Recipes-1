    Welcome to More-Recipes API Documentation

    Introduction

    More-Recipe API contains several API end points that allows users to signup, signin, add, retrieve, edit, and delete recipe. User can also upvote or downvote a recipe, add recipe to his/her favourite or remove from his/her favourite, get recipes with most upvote in ascending order. It also offers a way to ensure that only authorized users can perform certain operations.

    Development

    The application was developed with

    • NodeJs : for creating application server
    • Express : for routing
    • ReactJs : for front-end rendering
    • Postgresql : for application database, used with sequelize as the ORM

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of NodeJs equal or greater than v6.8.0.

    • Clone the repository from a terminal git clone https://github.com/chykehyman/More-Recipes.git
    • Move into the project directory cd More-Recipes
    • Add a .env file and add any preferred postgresql DB_URL.
    • Install project dependencies npm install
    • Start the express server npm start

    Json Web Token (JWT) for User Authentication

    This application uses username & password to authenticate users while generating a unique token. To use the token returned after successful signin or signup:

    • Save the generated token to the browser local storage or session storage with a property name "jwtToken". This gives users the priviledge to certain parts and aspect of the application they have permission to.

    • If using Postman to test the routes, copy the token from the login or sign up response to the headers with key name x-access-token.

    API Overview

    User

    EndPoint Functionality
    POST /api/v1/users/signup Creates a new user.
    POST /api/v1/users/signin Logs in a user.
    GET /api/v1/user/profile Retreives a logged in user's profile based on user decoded id.
    PUT /api/v1/user/profile Updates any logged in user's Profile except email based on the users id

    Recipe

    EndPoint Functionality
    POST /api/v1/recipes Creates a new recipe. (available only to authenticated users)
    GET /api/v1/recipes Gets all recipes. (available to everyone)
    GET /api/v1/recipes/:recipeID' Finds a recipe by id. (available to everyone)
    GET /api/v1/user/recipes Gets all recipes belonging to a particular user (available only to authenticated users)
    GET /api/v1/recipes?sort=upvotes&&order=desc Gets all recipes with the most up vote (available only to authenticated users)
    PUT /api/v1/recipes/:recipeID Updates a recipe. (available only to authenticated & users who added the recipe)
    DELETE /api/v1/recipes/:recipeID Deletes a recipe. (available only to the users who added the recipe)'

    Favourite

    EndPoint Functionality
    POST /api/v1/recipes/:recipeID/favorites Adds or removes a recipe to/from user's favourite recipes catalog. (available only to authenticated users)
    GET /api/v1/user/favorites Gets all user's favourite recipes. (available only to authenticated users)
    DELETE /api/user/favorites/:recipeID Delete a recipe from user's favourite recipes catalog. (available only to authenticated users)

    Voting

    EndPoint Functionality
    POST /api/v1/recipes/:recipeID/vote Upvoting and downvoting a recipe. (available only to authenticated users)

    Reviews

    EndPoint Functionality
    POST /api/recipes/:recipeID/reviews Post review for a recipe. (available only to authenticated users)

    Errors

    The More-Recipes API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    406 Not Acceptable -- You requested a format that isn't json.
    409 Conflicts -- Your request already exist.
    422 Unprocessable entity -- Your requested format was undefined.
    500 Internal Server Error -- We had a problem with our server. Try again later.
